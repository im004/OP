1. Programming Workflow & Design
   Top-Down Coding (The Blueprint): Write your main() function first using empty placeholder functions. Map out how data should flow before writing individual logic.

Incremental Development: Build and test one function at a time. Never write a 50-line program and hit run for the first time at the end.

Keep Math as Math: Never convert numerical inputs into strings (str()) if you plan to do math with them later. Keep them as float or int.

2. Data Structures: Normal vs. Curly Brackets
   Normal Brackets () → Tuples: Fixed, ordered shelves. You can only look up data using index numbers (e.g., e[0]). Passing text keys causes a TypeError.

Curly Brackets {} → Dictionaries: Labeled storage lockers. Data is stored in Key: Value pairs. You look up data using text labels (e.g., e["amount"]).

Case Sensitivity: Keys are strictly case-sensitive. e["Amount"] and e["amount"] are entirely different. Mismatches cause a KeyError.

3. Advanced Logic & Shorthand Syntax
   List Comprehension: A standard for loop compressed into one line.

Shorthand: sum(e["amount"] for e in expenses)

Longhand equivalent: Create a total = 0, loop through the list, add each amount to total.

Nested Loops (Loops inside Loops): Used when you need a "sub-task" to repeat perfectly before moving to the next main task.

Outer Loop: Collects multiple expenses until user types 'done'.

Inner Loop: Traps the user until they enter a valid, positive amount for that specific expense.

4. Defensive Coding & File Handling Syntax
   The Try-Except Security Guard: Protects programs from crashing due to unexpected user inputs (like typing letters instead of numbers).

val = float(input("Enter number: "))

## 🧭 1. Navigation & Inspection

* `pwd` $\rightarrow$ **Print Working Directory:** "Where am I right now?"
* `ls` $\rightarrow$ **List:** Lists visible files/folders in the current directory.
* `ls -la` $\rightarrow$ **Long & All:** Lists *everything*, including hidden configuration files (starting with `.`) and details like file size, owner, and permissions.
* `cd ~` $\rightarrow$ **Home:** Jumps back to your primary user profile directory.
* `cd ..` $\rightarrow$ **Up:** Jumps back one folder level (up to the parent directory).
* `cd -` $\rightarrow$ **Toggle:** Jumps back to the *previous* directory you were just in.

---

## 📂 2. File & Folder Manipulation

* `mkdir -p path/to/folder` $\rightarrow$ **Parents:** Creates nested folders all at once (e.g., builds `path`, then `to`, then `folder`).
* `touch filename.txt` $\rightarrow$ Creates a blank new file.
* `cp source.txt backup.txt` $\rightarrow$ **Copy:** Clones a file. Leaves the original completely untouched.
* `mv old.txt new.txt` $\rightarrow$ **Move / Rename:** Relocates or renames a file. **Destructive:** No copy remains of the original name/location.
* `rm filename.txt` $\rightarrow$ **Remove:** Permanently deletes a file. **Danger:** No recycling bin; it is gone instantly.

---

## 🔍 3. Reading & Searching Content

* `cat file.txt` $\rightarrow$ **Concatenate:** Dumps the *entire* content of a file into the terminal screen at once.
* `head -n 5 file.txt` $\rightarrow$ Shows only the **top** 5 lines of a file.
* `tail -n 5 file.txt` $\rightarrow$ Shows only the **bottom** 5 lines of a file (great for reading log files).
* `echo "text" >> file.txt` $\rightarrow$ **Append:** Appends a line of text to the bottom of a file. (Single `>` overwrites the whole file!).
* `grep "target" file.txt` $\rightarrow$ Searches a single file and prints lines containing the word "target".
* `grep -r "target" ~/folder/` $\rightarrow$ **Recursive:** Digs into the specified folder and *all* nested subfolders to find the word "target".

---

## ⛓️ 4. Chaining & Logic (Piping)

* The `|` (Pipe) key connects commands together.
* **The Blueprint:** `[Command A] | [Command B]`
* **How it works:** It takes the text output generated by Command A and feeds it directly as the input for Command B, skipping the need to save a temporary file.
* *Example:* `ls -la ~ | grep "Desktop"` (Lists *all* files in home, but only shows the line containing the word "Desktop").

---

## 🛡️ 5. Permissions & Scripting

* `#! /bin/bash` $\rightarrow$ **Shebang:** The very first line inside a script file. Tells the computer, *"Run the code below using the Bash terminal environment."*
* `./my_script.sh` $\rightarrow$ Tells the terminal to execute a script located in the current directory (`.`).
* **The Guardrail:** New files have **zero** execution rights by default. If you try to run them, the OS blocks it (`Permission denied`).
* `chmod +x script.sh` $\rightarrow$ **Change Mode:** Adds the "Execute" flag, authorizing the operating system to run the file as an application.

### The Permission Cheat-Code

Permissions are calculated using numbers: **Read (4), Write (2), Execute (1)**. You add them together for three groups: **Owner | Group | Everyone Else**.

* `chmod 755 file` $\rightarrow$ **Full/Safe:** Owner can do everything ($4+2+1=7$). Everyone else can only read and execute ($4+1=5$). Standard for scripts.
* `chmod 644 file` $\rightarrow$ **Standard Document:** Owner can read and write ($4+2=6$). Everyone else can only read ($4$). Standard for text files.
